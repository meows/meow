# Technology

- Drizzle ORM + PostgreSQL (postgres.js driver), dialect `postgresql`. Default casing (no `casing` option) — give columns explicit snake_case names in schema definitions, as the generated auth schema does.
- Consumed as TypeScript source by the apps (`#database/*` subpath imports through the `@kit/database` exports map) — no build step.
- `src/client.ts` is a pure factory: `createDb(url)` → `{ driver, db }`. No env access, no top-level connection (postgres.js connects lazily). Keep it that way — the better-auth CLI imports this module without a live database.

# Env

- `drizzle.config.ts` reads `process.env.DATABASE_URL` bare; drizzle-kit auto-loads `./.env`.
- Local dev DB options: Homebrew Postgres (`postgres://<macos-user>@localhost:5432/meow`) or `docker compose up -d` at the repo root (`postgres://meow:meow@localhost:5432/meow`).

# Scripts (uncached by design — they mutate state)

- `bun run db:up` / `bun run db:down` — root docker-compose Postgres up (waits for healthy) / down including the volume (dev DB is disposable)
- `bun run db:generate` — emit SQL migration from schema changes into `migrations/`
- `bun run db:migrate` — apply migrations
- `bun run db:reset` — drop `public` + drizzle's journal schema, re-apply all migrations (`src/script/reset.ts`)
- `bun run db:seed` — seed data via Drizzle (`src/script/seed.ts`; currently a mock — nothing to seed yet)
- `bun run db:studio` — Drizzle Studio

# Schema layout

- `src/schema/app.ts` — application tables (hand-written)
- `src/schema/auth.ts` — GENERATED by better-auth; never edit by hand. Regenerate from `server-api/`:
  `bunx @better-auth/cli generate --config src/auth.ts --output ../database/src/schema/auth.ts`
  (server-api's `.env` must be populated; no live DB needed). Run after any better-auth config/plugin change, then `db:generate` + `db:migrate`.
- `src/schema/index.ts` — barrel re-exporting both.
