# Technology

- Our build system is Vite+ (`vp dev`, `vp build`); the bundler is rolldown-vite with `@vitejs/plugin-react-oxc`.
- We use React with Tailwind v4 (theme lives in `src/global.css`, no tailwind config file).
- We use shadcn with Base UI (style `base-vega` in `components.json`) but other component libraries are ok.
- Lint/format come from the repo root (`vp check`) — Oxlint + Oxfmt, configured in the root `vite.config.ts`.

# Style

- We want the ergonomic standards of a library author.
- Prefer idiomatic React and TypeScript patterns.

# Aliases

- `~/…` → this app's `src/` (tsconfig `paths` + Vite alias).
- `@/…` → `src/component/` (shadcn convention; `@/ui/button` = `src/component/ui/button.tsx`).
- `#common/…` → `@kit/common` source (Node subpath imports; same spelling repo-wide).

# Constraint

- Resist modifying base shadcn components without a good argument.
- Don't modify global css without a good argument.
- We are doing single-page app (SPA) and don't care about SSR.
- Dev server proxies `/api` → `http://localhost:3000` (server-api); auth cookies are same-origin.
- `src/routeTree.gen.ts` is generated by the TanStack Router plugin — committed, but excluded from lint/fmt; never edit by hand.

# Directory Structure

- `components.json` shadcn configs (aliases: components `@`, ui `@/ui`, utils `~/lib/utils`)
- `tsconfig.json`
- `vite.config.ts` router plugin (before react-oxc), Tailwind, aliases, `/api` proxy
- 📁 `public` static assets (Vite default location, package root)
  - 📁 `pixel` pixel art
  - 📁 `sound`
- 📁 `src` source code
  - `env.ts` local client environment variables (Valibot over `import.meta.env`)
  - `global.css` Tailwind v4 theme config (colors, fonts, animations, keyframes)
  - `routeTree.gen.ts` generated route tree
  - 📁 `component` shared React components
    - 📁 `ui` shadcn components
    - 📁 `shell` application shell
    - 📁 `icon` artistic icon assets
  - 📁 `hook` React hooks
  - 📁 `lib`
    - `auth.ts` better-auth client
    - `utils.ts` shadcn utility function (cn)
  - 📁 `route` we're using Tanstack Router (`routesDirectory: './src/route'`)
    - `__root.tsx` root layout route
    - `index.tsx` index route
    - 📁 `app` the session protected portion of our app
    - 📁 `auth` registration, login, recovery stuff
  - 📁 `state` Jotai atoms, global state, etc.
