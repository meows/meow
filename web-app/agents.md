# Technology

- Our build system is Vite+ (`vp dev`, `vp build`); the bundler is rolldown-vite with `@vitejs/plugin-react-oxc`.
- We use React with Tailwind v4 (theme lives in `src/global.css`, no tailwind config file).
- We use shadcn with Base UI (style `base-vega` in `components.json`) but other component libraries are ok.
- Lint/format come from the repo root (`vp check`) вАФ Oxlint + Oxfmt, configured in the root `vite.config.ts`.

# Style

- We want the ergonomic standards of a library author.
- Prefer idiomatic React and TypeScript patterns.

# Aliases

- `~/вА¶` вЖТ this app's `src/` (tsconfig `paths` + Vite alias).
- `@/вА¶` вЖТ `src/component/` (shadcn convention; `@/ui/button` = `src/component/ui/button.tsx`).
- `#common/вА¶` вЖТ `@meow/common` source (Node subpath imports; same spelling repo-wide).

# Constraint

- Resist modifying base shadcn components without a good argument.
- Don't modify global css without a good argument.
- We are doing single-page app (SPA) and don't care about SSR.
- Dev server proxies `/api` вЖТ the server-api (port from Infisical `/network` `API_PORT`, default 3000); auth cookies are same-origin.
- `src/routeTree.gen.ts` is generated by the TanStack Router plugin вАФ committed, but excluded from lint/fmt; never edit by hand.

# Directory Structure

- `components.json` shadcn configs (aliases: components `@`, ui `@/ui`, utils `~/lib/utils`)
- `tsconfig.json`
- `vite.config.ts` router plugin (before react-oxc), Tailwind, aliases, `/api` proxy; reads dev host/ports from Infisical `/network` via `loadEnv`
- рЯУБ `public` static assets (Vite default location, package root)
  - рЯУБ `pixel` pixel art
  - рЯУБ `sound`
- рЯУБ `src` source code
  - `env.ts` local client environment variables (Valibot over `import.meta.env`)
  - `global.css` Tailwind v4 theme config (colors, fonts, animations, keyframes)
  - `routeTree.gen.ts` generated route tree
  - рЯУБ `component` shared React components
    - рЯУБ `ui` shadcn components
    - рЯУБ `shell` application shell
    - рЯУБ `icon` artistic icon assets
  - рЯУБ `hook` React hooks
  - рЯУБ `lib`
    - `auth.ts` better-auth client
    - `utils.ts` shadcn utility function (cn)
  - рЯУБ `route` we're using Tanstack Router (`routesDirectory: './src/route'`)
    - `__root.tsx` root layout route
    - `index.tsx` index route
    - рЯУБ `app` the session protected portion of our app
    - рЯУБ `auth` registration, login, recovery stuff
  - рЯУБ `state` Jotai atoms, global state, etc.
